## Стрингове и изход

### `echo` — отпечатване на текст

`echo` отпечатва текст в терминала. Прост на пръв поглед, но с важни детайли.

```bash
echo "Здравей, свят!"          # Здравей, свят!
echo Здравей                   # работи и без кавички
echo                           # отпечатва празен ред
echo -n "Без нов ред"          # -n: не добавя \n в края
echo -e "Ред 1\nРед 2"         # -e: интерпретира escape символи
echo -e "Колона1\tКолона2"     # \t = табулация
```

| Флаг | Значение |
|------|----------|
| `-n` | не добавя нов ред в края |
| `-e` | интерпретира `\n`, `\t`, `\\` и др. escape символи |
| `-E` | изключва интерпретацията (по подразбиране) |

```bash
# Escape символи с -e:
echo -e "\n--- Заглавие ---\n"   # празен ред преди и след
echo -e "Червено: \e[31mТекст\e[0m"  # ANSI цветове в терминала
```

> **`echo` vs `printf`:** За по-сложно форматиране е по-надеждно да се използва `printf` — поведението на `echo` с флагове се различава между различните shell-ове. `echo` е удобен за бързо отпечатване, `printf` за скриптове.


### Двойни кавички `""`

Двойните кавички **запазват буквалния смисъл на повечето символи**, но позволяват **заместване на променливи и команди**. Bash чете вътрешността и заменя `$VAR`, `$()` и `\` с реалните им стойности.

```bash
name="Ivan"
echo "Здравей, $name!"         # Здравей, Ivan!
echo "Домашната ти директория е $HOME"   # /home/ivan
echo "Сега е $(date)"          # Сега е Tue Mar 17 10:22:00 2026
echo "Цена: \$5"               # Цена: $5  (\ escape-ва $)
```

### Единични кавички `''`

Единичните кавички са **буквални** — всичко вътре се третира като чист текст. Никакво заместване, никакви специални символи. Дори `\` няма специален смисъл.

```bash
echo 'Здравей, $name!'         # Здравей, $name!  (не замества)
echo 'Домашната ти директория е $HOME'   # $HOME (буквално)
echo 'Сега е $(date)'          # Сега е $(date)  (не изпълнява)
echo 'Backslash: \'            # грешка — не можеш да escape-неш ' с '
```

### Правилото

| Ситуация | Кавички |
|----------|---------|
| Искаш да замести променливи | `"двойни"` |
| Искаш буквален текст, без заместване | `'единични'` |
| Стрингът съдържа единични кавички | `"двойни"` |
| Стрингът съдържа специални shell символи | `'единични'` |

```bash
echo "It's a test"             # работи — единична кавичка вътре в двойни
echo 'Say "hello"'             # работи — двойни вътре в единични
echo "Цена: \$5.00"            # escape с \ вътре в двойни
```

### Операции с низове

```bash
str="Hello, World"

echo ${#str}                   # 12 — дължина на стринга
echo ${str:7}                  # World — от позиция 7 нататък
echo ${str:7:5}                # World — от позиция 7, 5 символа
echo ${str/World/Bash}         # Hello, Bash — замества първото съвпадение
echo ${str//l/L}               # HeLLo, WorLd — замества всички съвпадения
echo ${str,,}                  # hello, world — всичко малки букви
echo ${str^^}                  # HELLO, WORLD — всичко главни букви

path="/home/ivan/notes.txt"
echo ${path##*/}               # notes.txt — само името на файла (след последния /)
echo ${path%/*}                # /home/ivan  — директорията (преди последния /)
echo ${path##*.}               # txt — само разширението
```

### Стойности по подразбиране

```bash
# Ако $name не е зададено или е празно — използвай стойност по подразбиране
echo ${name:-"Гост"}           # показва "Гост" ако $name е празно
echo ${name:="Гост"}           # задава $name="Гост" ако е празно
echo ${name:?"Грешка: name не е зададено"}  # показва грешка ако е празно
```

## Заместване

### `$()` — Command Substitution

`$()` изпълнява команда и **замества израза с нейния изход**. Можеш да използваш резултата от всяка команда навсякъде — в присвоявания, в стрингове, като аргументи.

```bash
echo "Днес е $(date +%A)"              # Днес е Tuesday
echo "Потребителят е $(whoami)"        # Потребителят е ivan
echo "Файлове: $(ls | wc -l)"          # Файлове: 42

# Присвояване
today=$(date +%Y-%m-%d)
files=$(find . -name "*.log" | wc -l)
current_dir=$(pwd)

# Като аргумент на друга команда
mkdir "backup_$(date +%Y%m%d)"        # backup_20260317
cat $(find . -name "*.conf")          # прочита всички .conf файлове
```

Вложено заместване — `$()` може да се влага:
```bash
echo "Последен файл: $(ls -t $(pwd) | head -1)"
```

> **`$()` vs `` ` `` `` ` ``:** Старият синтаксис за command substitution са обратни кавички `` `команда` ``. Двете са еквивалентни, но `$()` е препоръчван — по-лесен за четене и влагане.

### `$(())` — Arithmetic Substitution

`$(())` изчислява **аритметични изрази** с цели числа и замества резултата.

```bash
echo $((3 + 5))                # 8
echo $((10 * 4 - 2))           # 38
echo $((10 / 3))               # 3  (целочислено деление!)
echo $((10 % 3))               # 1  (остатък)
echo $((2 ** 8))               # 256 (степенуване)

a=10
b=3
echo $((a + b))                # 13
echo $((a * b))                # 30

# Инкремент
count=0
count=$((count + 1))
echo $count                    # 1

# В стрингове
echo "5 на квадрат е $((5**2))"     # 5 на квадрат е 25
echo "RAM: $(($(free -m | awk 'NR==2{print $2}') )) MB"
```

> **Важно:** `$(())` работи само с **цели числа**. За десетични числа се използва `bc` или `awk`:
```bash
echo "scale=2; 10 / 3" | bc        # 3.33
awk 'BEGIN { print 10 / 3 }'       # 3.33333
```

## Помощни команди

### `sort` — сортиране

`sort` сортира редовете на текстов вход. По подразбиране — лексикографски (азбучен ред).

```bash
sort file.txt                  # сортира азбучно
sort -r file.txt               # обратен ред
sort -n numbers.txt            # числово сортиране (не лексикографско)
sort -u file.txt               # премахва дубликатите (unique)
sort -k2 file.txt              # сортира по второто поле
sort -k2 -n file.txt           # сортира числово по второто поле
sort -t: -k3 -n /etc/passwd    # сортира по 3-то поле, разделено с ':'
```

| Флаг | Значение |
|------|----------|
| `-r` | обратен ред |
| `-n` | числово сортиране |
| `-u` | само уникални редове |
| `-k <N>` | сортира по N-то поле |
| `-t <sep>` | задава разделителя на полетата |
| `-f` | игнорира главни/малки букви |
| `-h` | сортира четими размери (1K, 2M, 3G) |
| `-R` | случаен ред (разбъркване) |

```bash
# Практически примери
du -sh * | sort -h             # сортира файлове по размер (четимо)
cat access.log | sort -k9 -n  # сортира лог по статус код
sort -t, -k2 data.csv          # сортира CSV по втора колона
```

### `tr` — трансформация на символи

`tr` (**tr**anslate) заменя или изтрива отделни символи. Работи само с **отделни символи**, не с низове.

```bash
echo "hello" | tr 'a-z' 'A-Z'         # HELLO — малки в главни
echo "HELLO" | tr 'A-Z' 'a-z'         # hello — главни в малки
echo "hello world" | tr ' ' '_'       # hello_world — интервали в _
echo "hello" | tr -d 'l'              # heo — изтрива всички 'l'
echo "aabbcc" | tr -s 'a-z'           # abc — squeeze (премахва повтарящи се)
echo "abc123" | tr -d '0-9'           # abc — изтрива цифрите
```

| Флаг | Значение |
|------|----------|
| `-d` | delete — изтрива символите |
| `-s` | squeeze — свива повторенията в едно |
| `-c` | complement — работи с всичко ОСВЕН посочените символи |

```bash
# Полезни класове символи
echo "Hello 123!" | tr -d '[:digit:]'      # Hello !  (изтрива цифри)
echo "Hello 123!" | tr -d '[:alpha:]'      #  123!    (изтрива букви)
echo "Hello 123!" | tr -d '[:space:]'      # Hello123! (изтрива интервали)
echo "Hello\tWorld" | tr '\t' ' '          # замества tab с интервал
```

### `cut` — изрязване на колони

`cut` извлича конкретни **полета (колони)** или **символи** от всеки ред.

```bash
# По разделител (-d) и поле (-f)
echo "ivan:1000:developers" | cut -d: -f1         # ivan
echo "ivan:1000:developers" | cut -d: -f2         # 1000
echo "ivan:1000:developers" | cut -d: -f1,3       # ivan:developers
echo "ivan:1000:developers" | cut -d: -f2-        # 1000:developers (от 2 до края)

# По позиция на символи (-c)
echo "Hello, World!" | cut -c1-5                  # Hello
echo "Hello, World!" | cut -c8-                   # World!
```

| Флаг | Значение |
|------|----------|
| `-d <sep>` | задава разделителя (по подразбиране: tab) |
| `-f <N>` | избира поле N (или диапазон: `1-3`, или `2-`) |
| `-c <N>` | избира символи по позиция |

```bash
# Практически примери
cut -d: -f1 /etc/passwd              # само потребителските имена
cut -d, -f2,4 data.csv               # 2-ра и 4-та колона от CSV
ls -l | cut -c1-10                   # само правата от ls -l
```

> **`cut` vs `awk`:** `cut` е по-бърз и по-прост за еднократно извличане на колони. `awk` е по-мощен когато трябва обработка или логика върху данните.

## `xargs` — превръщане на изход в аргументи
 
`xargs` решава един фундаментален проблем в терминала: повечето команди не четат от стандартния вход (`stdin`) — те очакват аргументи. `xargs` взима редовете от `stdin` и ги **превръща в аргументи** на друга команда.
 
#### Защо е нужен
 
```bash
# Проблемът: find намери файловете, но rm не чете от stdin
find . -name "*.tmp" | rm          # ⚠️  не работи — rm не чете от pipe
 
# Решението с xargs:
find . -name "*.tmp" | xargs rm    # ✅  xargs взима пътищата и ги подава на rm
```
 
Без `xargs` единствената алтернатива е `-exec` при `find`, но `xargs` е по-гъвкав и работи с всяка команда.
 
#### Как работи
 
`xargs` чете `stdin` ред по ред, натрупва аргументите и извиква командата — по подразбиране с **всички аргументи наведнъж** в края:
 
```
find . -name "*.log"    →    изход:
                              ./app.log
                              ./error.log
                              ./access.log
                                   │
                                   ▼ xargs rm
                         rm ./app.log ./error.log ./access.log
                         └── една команда с три аргумента
```
 
Сравни с `-exec ... \;` при `find`, което извиква командата поотделно за всеки файл — `xargs` е значително по-бързо при много файлове.
 
#### Основна употреба
 
```bash
find . -name "*.tmp" | xargs rm                  # изтрива всички .tmp файлове
find . -name "*.jpg" | xargs ls -lh              # показва детайли за снимките
find . -type f -name "*.py" | xargs wc -l        # брои редовете в .py файловете
echo "file1.txt file2.txt" | xargs cat           # показва съдържанието на двата файла
```
 
#### Флагове
 
| Флаг | Значение |
|------|----------|
| `-I {}` | задава placeholder — `{}` се замества с всеки аргумент поотделно |
| `-n <N>` | подава максимум N аргумента на извикване |
| `-P <N>` | паралелно изпълнение с N процеса едновременно |
| `-0` | аргументите са разделени с `\0` вместо newline (за имена с интервали) |
| `-t` | verbose — отпечатва командата преди изпълнение |
| `-p` | интерактивен — пита за потвърждение преди всяко извикване |
| `--no-run-if-empty` | не извиква командата ако stdin е празен |
 
#### `-I {}` — placeholder за позициониране
 
По подразбиране `xargs` добавя аргументите в **края** на командата. С `-I {}` можеш да контролираш точно **къде** се поставят:
 
```bash
# Без -I {} — аргументът отива в края
find . -name "*.conf" | xargs cp /backup/
# → cp /backup/ file1.conf file2.conf  ⚠️  грешен ред на аргументите
 
# С -I {} — {} се замества с аргумента
find . -name "*.conf" | xargs -I {} cp {} /backup/
# → cp file1.conf /backup/
# → cp file2.conf /backup/
 
# Примери с позициониране
find . -name "*.log" | xargs -I {} mv {} {}.bak        # преименува: app.log → app.log.bak
find . -name "*.txt" | xargs -I {} sh -c 'echo "=== {} ===" && cat {}'
ls *.jpg | xargs -I {} convert {} thumbnails/thumb_{}  # ImageMagick thumbnail
```
 
#### `-n` — брой аргументи на извикване
 
```bash
echo "a b c d e f" | xargs -n 2 echo
# echo a b
# echo c d
# echo e f
 
find . -name "*.txt" | xargs -n 1 wc -l    # wc за всеки файл поотделно
find . -name "*.txt" | xargs -n 3 cat      # cat на по 3 файла наведнъж
```
 
#### `-P` — паралелно изпълнение
 
```bash
# Компресира файловете паралелно с 4 процеса едновременно
find . -name "*.log" -mtime +30 | xargs -P 4 -I {} gzip {}
 
# Изтегля URL-и паралелно
cat urls.txt | xargs -P 8 -I {} curl -O {}
```
 
#### `-0` — безопасна работа с имена, съдържащи интервали
 
Стандартно `xargs` разделя аргументите по **интервали и нови редове** — файл с интервал в името се разбива на два аргумента и командата се проваля. Решението е `find -print0` и `xargs -0`, които ползват нулевия байт (`\0`) като разделител:
 
```bash
# Проблем — файл "my document.txt" се разбива
find . -name "*.txt" | xargs rm                  # ⚠️  "my" и "document.txt" се третират отделно
 
# Решение — нулев байт като разделител
find . -name "*.txt" -print0 | xargs -0 rm       # ✅  работи с всякакви имена
find . -name "*.txt" -print0 | xargs -0 -I {} cp {} /backup/
```
 
> **Добра практика:** Винаги използвай `-print0 | xargs -0` когато обработваш файлове — никога не знаеш дали имената съдържат интервали, скоби или Unicode символи.
 
#### `xargs` vs `find -exec`
 
И двата подхода вършат едно и също, но с различни компромиси:
 
| | `xargs` | `find -exec ... \;` | `find -exec ... +` |
|---|---------|---------------------|---------------------|
| Скорост | ✅ бърз | ❌ бавен (1 процес/файл) | ✅ бърз |
| Паралелност | ✅ с `-P` | ❌ | ❌ |
| Имена с интервали | с `-0` | ✅ винаги | ✅ винаги |
| Работи с всяка команда | ✅ | ✅ | ✅ |
| Placeholder позиция | ✅ с `-I {}` | ✅ с `{}` | само в края |
 
```bash
# Еквивалентни команди:
find . -name "*.tmp" -exec rm {} +          # find с -exec
find . -name "*.tmp" | xargs rm             # find с xargs
 
# xargs е по-удобен когато имаш готов списък (не само от find)
cat files_to_delete.txt | xargs rm
```
 
#### Практически примери
 
```bash
# Намери и изтрий всички node_modules директории
find . -type d -name "node_modules" -print0 | xargs -0 rm -rf
 
# Брои редовете на всички Python файлове в проект
find . -name "*.py" -print0 | xargs -0 wc -l | sort -n
 
# Замести "localhost" с "127.0.0.1" в всички конфигурационни файлове
find . -name "*.conf" -print0 | xargs -0 sed -i 's/localhost/127.0.0.1/g'
 
# Паралелно изтегляне на файлове от списък
cat download_list.txt | xargs -P 5 -I {} wget -q {}
 
# Намери дублирани файлове по MD5
find . -type f -print0 | xargs -0 md5sum | sort | uniq -d -w 32
```

## Обработка на текст

### `grep` — търсене по шаблон

`grep` (**G**lobal **R**egular **E**xpression **P**rint) търси редове, съвпадащи с даден шаблон. Един от най-използваните инструменти в терминала.

#### Как работи

`grep` чете входа **ред по ред**. За всеки ред проверява дали шаблонът съвпада някъде в него — ако да, отпечатва целия ред. Ако не — редът се пропуска.

```
Вход (ред по ред)          Шаблон: "error"        Изход
─────────────────────      ──────────────         ──────────────────────
"[INFO] server started"  → няма съвпадение   →   (пропуснат)
"[ERROR] connection lost" → съвпадение       →   "[ERROR] connection lost"
"[DEBUG] retrying..."    → няма съвпадение   →   (пропуснат)
"[ERROR] timeout"        → съвпадение        →   "[ERROR] timeout"
```

```bash
grep "error" log.txt              # редове съдържащи "error"
grep -i "error" log.txt           # без значение главни/малки букви
grep -v "debug" log.txt           # редовете, които НЕ съдържат "debug"
grep -n "error" log.txt           # показва номерата на редовете
grep -c "error" log.txt           # само броят на съвпадащите редове
grep -r "TODO" ./src/             # рекурсивно в директория
grep -l "error" *.log             # само имената на файловете (не съдържанието)
grep -w "log" file.txt            # само цялата дума "log" (не "logger")
```
>Важно: `grep` търси **съдържащ**, не **равен** — `grep "log"` ще намери и `"logger"`, и `"catalog"`, освен ако не ограничиш с `-w`.


| Флаг | Значение |
|------|----------|
| `-i` | case-insensitive |
| `-v` | invert — редовете без съвпадение |
| `-n` | показва номера на реда |
| `-c` | брои съвпаденията |
| `-r` | рекурсивно в директории |
| `-l` | само имената на файловете |
| `-w` | само цели думи |
| `-A <N>` | показва N реда след съвпадението |
| `-B <N>` | показва N реда преди съвпадението |
| `-C <N>` | показва N реда преди И след (контекст) |
| `-E` | разширени регулярни изрази (като `egrep`) |
| `-o` | показва само съвпадащата част, не целия ред |

#### Регулярни изрази (regex)
 
Шаблоните в `grep` могат да бъдат обикновен текст или **регулярни изрази** — специален синтаксис за описване на текстови шаблони. С флага `-E` се отключват разширените regex (ERE).

[Повече информация за регулярните изрази.](./Regex/README.md)

```bash
# Контекст около съвпадението
grep -C 3 "ERROR" app.log         # 3 реда преди и след всяка грешка

# Регулярни изрази
grep "^ERROR" log.txt             # редове, започващи с "ERROR"
grep "\.txt$" file.txt            # редове, завършващи с ".txt"
grep -E "error|warning" log.txt   # "error" ИЛИ "warning"
grep -E "[0-9]{3}" log.txt        # точно 3 цифри
grep -E "^[A-Z]" file.txt         # редове, започващи с главна буква

# В комбинация с pipe
cat access.log | grep "404" | wc -l          # брой 404 грешки
ps aux | grep nginx                           # търси nginx процес
cat /etc/passwd | grep -v "^#" | cut -d: -f1 # потребителите без коментари
```


### `sed` — редактиране на потоци

`sed` (**S**tream **ED**itor) обработва текст ред по ред, прилагайки команди за трансформация. Не редактира файлове директно (по подразбиране) — чете вход, трансформира и отпечатва.

#### Как работи

`sed` има **вътрешен цикъл** — той работи с два буфера: **pattern space** (работното пространство) и **hold space** (дългосрочна памет). За всеки ред от входа:

```
1. Чете реда и го поставя в pattern space
2. Изпълнява всички команди върху pattern space
3. Отпечатва pattern space (освен ако не е подаден -n)
4. Изчиства pattern space и преминава към следващия ред
```

```
Вход           pattern space        Команда: s/foo/bar/g       Изход
────────        ─────────────        ──────────────────────     ──────
"foo baz"   →   "foo baz"        →   замества → "bar baz"   →  "bar baz"
"hello"     →   "hello"          →   няма съвпадение         →  "hello"
"foo foo"   →   "foo foo"        →   замества → "bar bar"   →  "bar bar"
```

#### Основен синтаксис

```bash
sed 'команда' файл
sed -e 'команда1' -e 'команда2' файл    # множество команди
sed -i 'команда' файл                   # редактира файла директно (in-place)
sed -i.bak 'команда' файл               # in-place, запазва backup с .bak
```

#### Заместване `s///`

Най-използваната команда — `s/шаблон/замяна/флагове`.

```bash
sed 's/foo/bar/' file.txt              # замества ПЪРВОТО "foo" на всеки ред
sed 's/foo/bar/g' file.txt             # замества ВСИЧКИ "foo" (g = global)
sed 's/foo/bar/i' file.txt             # без значение главни/малки букви
sed 's/foo/bar/2' file.txt             # замества само второто съвпадение
sed 's/foo/bar/gI' file.txt            # всички + case-insensitive

# Разделителят не е задължително /
sed 's|/old/path|/new/path|g' file.txt  # удобно при пътища
sed 's#foo#bar#g' file.txt              # работи с всеки символ
```

**Capture groups** — можеш да "захванеш" части от шаблона с `\(...\)` и да ги върнеш в замяната чрез `\1`, `\2`:

```bash
# Размени местата на две думи
echo "John Smith" | sed 's/\(.*\) \(.*\)/\2, \1/'   # Smith, John

# Обгради числата в скоби
echo "код 404 грешка" | sed 's/[0-9]\+/(&)/g'        # код (404) грешка
# & означава "цялото съвпадение"

# Добави http:// към URL-и без протокол
sed 's/^\(www\.\)/http:\/\/\1/' file.txt
```

#### Изтриване на редове `d`

```bash
sed '3d' file.txt                      # изтрива 3-ти ред
sed '3,7d' file.txt                    # изтрива редове 3 до 7
sed '/pattern/d' file.txt              # изтрива редовете съдържащи pattern
sed '/^#/d' file.txt                   # изтрива коментари (редове с #)
sed '/^$/d' file.txt                   # изтрива празни редове
sed '/^[[:space:]]*$/d' file.txt       # изтрива редове само с интервали
```

#### Принтиране `p`

```bash
sed -n '5p' file.txt                   # показва само 5-ти ред (-n спира стандартния изход)
sed -n '5,10p' file.txt                # показва редове 5 до 10
sed -n '/error/p' file.txt             # показва само редовете с "error"
```

#### Адресиране

Адресирането казва на `sed` кои редове да обработва. Без адрес — командата се прилага към всички редове.

```bash
sed '2s/foo/bar/' file.txt             # замества само на ред 2
sed '2,5s/foo/bar/' file.txt           # само на редове 2-5
sed '/start/,/end/s/foo/bar/' file.txt # между редовете съдържащи start и end
sed '$s/foo/bar/' file.txt             # само на последния ред
sed '~2s/foo/bar/' file.txt            # на всеки втори ред (0, 2, 4…)
```

#### In-place редактиране

```bash
sed -i 's/localhost/127.0.0.1/g' config.txt           # директна промяна
sed -i.bak 's/localhost/127.0.0.1/g' config.txt       # с backup
sed -i '/^#/d' config.txt                              # изтрива коментарите
```

```bash
# Практически примери
sed 's/^/  /' file.txt                 # добавя 2 интервала в началото на всеки ред
sed 's/ *$//' file.txt                 # премахва trailing whitespace
sed -n '10,20p' large_file.txt         # показва само редове 10-20 (по-бързо от head/tail)
sed 's/[0-9]\+/NUM/g' file.txt         # замества всички числа с "NUM"
```



### `awk` — обработка на структурирани данни

`awk` е малък програмен език, специализиран в обработката на текст, организиран в **полета и редове**. Изключително мощен за работа с табулирани данни — може да филтрира, трансформира, сумира и форматира без да пишеш отделна програма.

#### Как работи

`awk` чете входа **ред по ред**. Всеки ред автоматично се разделя на **полета** по разделител (по подразбиране: всяка последователност от интервали/табове). Полетата са достъпни като `$1`, `$2`, `$3`…

Програмата в `awk` е изградена от блокове с формат `шаблон { действие }`. За всеки ред `awk` проверява шаблона — ако съвпада, изпълнява действието. Ако шаблонът е пропуснат — действието се изпълнява за всеки ред.

```
Програма:  шаблон { действие }
           ───────  ──────────
           кога?    какво?

Примери:
  { print $1 }           → за ВСЕКИ ред, отпечатай първото поле
  NR > 1 { print $0 }    → само ако номерът на реда е > 1
  /error/ { print $0 }   → само ако редът съдържа "error"
  $3 > 100 { print $1 }  → само ако третото поле е > 100
```

```
Вход:  "ivan   25  developer"
        │       │   │
        $1      $2  $3          $0 = целият ред
        NF = 3  (три полета)    NR = 1 (първи ред)
```

```bash
echo "ivan 25 developer" | awk '{print $1}'     # ivan
echo "ivan 25 developer" | awk '{print $3}'     # developer
echo "ivan 25 developer" | awk '{print $2, $1}' # 25 ivan
```

#### Специални променливи

| Променлива | Значение |
|------------|----------|
| `$0` | целият текущ ред |
| `$1, $2…` | полета по ред |
| `NF` | **N**umber of **F**ields — броят на полетата в текущия ред |
| `NR` | **N**umber of **R**ecords — номерът на текущия ред (от началото) |
| `FNR` | номерът на реда в текущия файл (при множество файлове) |
| `FS` | **F**ield **S**eparator — разделителят на полетата (по подр.: интервал) |
| `OFS` | **O**utput **F**ield **S**eparator — разделителят в изхода |
| `RS` | **R**ecord **S**eparator — разделителят на редовете (по подр.: \n) |
| `ORS` | **O**utput **R**ecord **S**eparator |

```bash
echo "ivan 25 developer" | awk '{print NF}'     # 3 — три полета
echo "ivan 25 developer" | awk '{print $NF}'    # developer — последното поле
echo "ivan 25 developer" | awk '{print $(NF-1)}'  # 25 — предпоследното поле
```

#### BEGIN и END

`BEGIN` и `END` са специални шаблони — не съвпадат с редове от входа, а с **моменти от изпълнението**. `BEGIN` се изпълнява веднъж преди четенето на първия ред — използва се за инициализация на променливи и задаване на разделители. `END` се изпълнява веднъж след последния ред — идеален за обобщения и финални изчисления.

```
BEGIN  →  { инициализация }
           ↓
ред 1  →  { обработка }
ред 2  →  { обработка }
  ...
ред N  →  { обработка }
           ↓
END    →  { обобщение, финален изход }
```

```bash
awk 'BEGIN { print "Начало" } { print $0 } END { print "Край" }' file.txt

# Брои редовете (като wc -l)
awk 'END { print NR }' file.txt

# Сумира числата в първата колона
awk '{ sum += $1 } END { print "Сума:", sum }' numbers.txt

# Среден брой думи на ред
awk '{ total += NF } END { print "Средно:", total/NR, "думи/ред" }' file.txt
```

#### Разделител на полетата

```bash
awk -F: '{print $1}' /etc/passwd               # : като разделител
awk -F, '{print $2}' data.csv                  # , за CSV
awk -F'\t' '{print $3}' data.tsv               # tab разделител
awk 'BEGIN{FS=":"} {print $1}' /etc/passwd     # FS в BEGIN блока
```

#### Условия и филтриране

```bash
awk '$3 > 100' file.txt                        # редове, където $3 > 100
awk '$1 == "ivan"' file.txt                    # само редове с ivan в $1
awk 'NR > 1' file.txt                          # пропуска първия ред (header)
awk 'NR % 2 == 0' file.txt                     # само четните редове
awk '/error/' file.txt                         # редове съдържащи "error" (като grep)
awk '!/debug/' file.txt                        # редове без "debug"
awk 'NR >= 5 && NR <= 10' file.txt             # редове 5 до 10
```

#### Форматиран изход с printf

```bash
awk '{printf "%-20s %5d\n", $1, $2}' file.txt
#    └── %-20s: стринг, 20 символа, ляво наредено
#               └── %5d: число, 5 символа, дясно наредено

awk '{printf "%s\t%s\n", $1, $3}' file.txt    # tab разделени колони
```

#### Практически примери

```bash
# Покажи само потребителите от /etc/passwd с UID >= 1000
awk -F: '$3 >= 1000 {print $1, $3}' /etc/passwd

# Намери най-голямата стойност в колона
awk 'BEGIN{max=0} $1>max{max=$1} END{print "Макс:", max}' numbers.txt

# Брои уникалните стойности в първата колона
awk '{count[$1]++} END {for (k in count) print k, count[k]}' file.txt

# Сумира размерите от ls -l
ls -l | awk 'NR>1 {sum += $5} END {print "Общо:", sum, "байта"}'

# Промени разделителя от : на ,
awk -F: 'BEGIN{OFS=","} {print $1,$3,$6}' /etc/passwd

# Покажи дублираните редове
awk 'seen[$0]++' file.txt

# Покажи редове между два шаблона
awk '/START/,/END/' file.txt
```

#### Сравнение на инструментите

| Инструмент | Добър за |
|------------|----------|
| `grep` | намиране и филтриране на редове по шаблон |
| `sed` | заместване, изтриване и трансформация на текст |
| `cut` | бързо извличане на колони по фиксиран разделител |
| `awk` | обработка на структурирани данни, аритметика, обобщения |

Трите инструмента покриват различни нужди и в реална работа почти винаги се използват заедно в pipeline. Данните текат от ляво на дясно — изходът на всяка команда става входът на следващата:

```bash
cat access.log | grep "POST" | awk '{print $7}' | sort | uniq -c | sort -rn | head -10

# Стъпки:
# 1. grep    — филтрира само POST заявките
# 2. awk     — извлича само URL-а (7-мо поле)
# 3. sort    — сортира URL-ите азбучно (нужно преди uniq)
# 4. uniq -c — брои колко пъти се появява всеки URL
# 5. sort -rn — сортира числово в обратен ред (най-много отгоре)
# 6. head -10 — взима само топ 10
# Резултат: Топ 10 POST endpoints по брой заявки
```

# Vim — Текстов редактор

## Откъде идва Vim?

За да разберем Vim, трябва да се върнем назад — в епохата преди графичните интерфейси.

През **1969 г.** Ken Thompson създава `ed` — първия текстов редактор за Unix. `ed` е изцяло **линеен** редактор: работи с команди на ред по ред, без да показва текста на екрана. Изглеждало е нещо такова:

```
$ ed file.txt
156                       ← ed съобщава броя байтове
1,3p                      ← ти искаш да видиш редове 1 до 3
this is line one
this is line two
this is line three
q                         ← изход
```

Нямало е курсор. Нямало е визуализация. Просто команди и резултати.

През **1976 г.** Bill Joy (по-късно съосновател на Sun Microsystems) пише `vi` (**vi**sual editor) — редактор, който за пръв път показва съдържанието на файла директно на екрана и позволява навигация с клавишите. По онова време дори това е революция. `vi` се превръща в стандарт и е включен в почти всяка Unix система до днес.

През **1991 г.** Bram Moolenaar издава **Vim** (**Vi** i**m**proved) — подобрена версия на `vi` с добавени функции: синтактично оцветяване, поддръжка на плъгини, многоредово отмяна, макроси и много повече. Vim бързо измества оригиналния `vi` и се превръща в един от най-популярните текстови редактори в света — позиция, която задържа и до днес.

---

## Защо е измислен и защо се използва днес?

Vim е проектиран за **ефективност при работа без мишка**. В ерата на терминалите мишката просто не е съществувала — всичко е ставало с клавиатурата. Vim е изграден около идеята, че ръцете трябва да напускат клавиатурата възможно най-рядко.

Днес Vim се използва по няколко причини:

**Присъства навсякъде.** Всеки Linux сървър, всеки macOS и повечето Unix системи имат `vi` или `vim` инсталиран по подразбиране. Когато се свържеш по SSH към отдалечена машина и трябва да редактираш конфигурационен файл — Vim е там. Nano може да го няма. VS Code определено го няма.

**Работи изцяло в терминала.** Не изисква графичен интерфейс — можеш да го използваш дори на сървър без GUI, или в tmux сесия.

**Скорост след научаването.** Vim командите са проектирани така, че честите операции да изискват минимум натискания. Редактирането се превръща в своеобразен "език" — комбинации от глаголи и обекти: `d3w` (изтрий 3 думи), `ci"` (промени съдържанието в кавички), `gg=G` (форматирай целия файл).

**Конфигурируем до краен предел.** `~/.vimrc` файлът позволява да настроиш всеки аспект на поведението. Съществуват стотици плъгини. Много съвременни редактори (VS Code, IntelliJ, Sublime) имат Vim режим именно защото потребителите го искат.

---

## Как се отваря Vim

```bash
vim                          # отваря празен буфер
vim file.txt                 # отваря съществуващ файл (или създава нов)
vim file1.txt file2.txt      # отваря няколко файла
vim +42 file.txt             # отваря файла и позиционира на ред 42
vim +/pattern file.txt       # отваря и търси шаблон
vim -R file.txt              # read-only режим
```

## Първото нещо, което трябва да знаеш

Vim **не е обикновен редактор**. Когато отвориш файл в Notepad или VS Code, можеш веднага да пишеш. Vim не работи така.

Vim има **режими** — в зависимост от активния режим едни и същи клавиши вършат напълно различни неща. Това е коренната разлика и причината Vim изглежда толкова объркващ в началото.

```
Отвориш vim → влизаш в Normal режим
                    │
                    ├── натиснеш i → Insert режим (пишеш текст)
                    │        └── натиснеш Esc → обратно в Normal
                    │
                    ├── натиснеш v → Visual режим (избираш текст)
                    │        └── натиснеш Esc → обратно в Normal
                    │
                    └── натиснеш : → Command режим (изпълняваш команди)
                             └── натиснеш Enter/Esc → обратно в Normal
```

**Normal режимът е "домашната база"** — там се връщаш след всяко действие. Ако се изгубиш, натисни `Esc` (или два пъти `Esc`) и ще се върнеш в Normal.

## Режим Normal

**Normal режимът** е активен при стартиране. Клавишите не въвеждат текст — те са **команди за навигация и редактиране**. Това е режимът, в който прекарваш по-голямата част от времето.

### Навигация

```
h  j  k  l   →   ляво, долу, горе, дясно  (вместо стрелките)
```

```
w             →   към началото на следващата дума
b             →   към началото на предишната дума
e             →   към края на следващата дума
W  B  E       →   същото, но само за ДУМИ (разделени по интервал)

0             →   начало на ред (буквално)
^             →   първият непразен символ на реда
$             →   край на ред
gg            →   начало на файла
G             →   край на файла
42G           →   ред 42
Ctrl+d        →   половин страница надолу
Ctrl+u        →   половин страница нагоре
Ctrl+f        →   цяла страница напред
Ctrl+b        →   цяла страница назад
```

### Редактиране

Командите в Normal режим следват структурата **[брой] глагол [обект]**:

```
d             →   delete (изтриване)
c             →   change (изтриване + влизане в Insert)
y             →   yank (копиране)
p             →   paste (поставяне след курсора)
P             →   paste (поставяне преди курсора)
```

Обектите описват върху какво действа командата:

```
w             →   дума (word)
b             →   предишна дума
$             →   до края на реда
0             →   до началото на реда
gg            →   до началото на файла
G             →   до края на файла
```

Комбинации:

```
dw            →   изтрива следващата дума
d$            →   изтрива до края на реда
dd            →   изтрива целия ред
3dd           →   изтрива 3 реда
yy            →   копира целия ред
5yy           →   копира 5 реда
cc            →   изтрива реда и влиза в Insert
D             →   изтрива до края на реда (= d$)
C             →   изтрива до края и влиза в Insert (= c$)
x             →   изтрива символа под курсора
r<символ>     →   заменя символа под курсора с нов
```

### Отмяна и повторение

```
u             →   undo (отмяна на последното действие)
Ctrl+r        →   redo (повторение на отмененото)
.             →   повтаря последната промяна — изключително мощна команда
```

### Търсене

```
/шаблон       →   търси напред
?шаблон       →   търси назад
n             →   следващото съвпадение
N             →   предишното съвпадение
*             →   търси думата под курсора (напред)
#             →   търси думата под курсора (назад)
```

## Режим Insert

**Insert режимът** е единственият режим, в който пишеш текст нормално. Влизаш в него от Normal режим с различни клавиши — изборът на клавиш определя откъде ще започнеш да пишеш:

```
i             →   Insert преди курсора
a             →   Insert след курсора (append)
I             →   Insert в началото на реда
A             →   Insert в края на реда
o             →   отваря нов ред под текущия и влиза в Insert
O             →   отваря нов ред над текущия и влиза в Insert
s             →   изтрива символа под курсора и влиза в Insert
S             →   изтрива целия ред и влиза в Insert
```

Докато си в Insert режим, в долния ляв ъгъл на екрана виждаш `-- INSERT --`.

За да излезеш от Insert режим и се върнеш в Normal: `Esc` или `Ctrl+[`.

> **Съвет:** Не оставай в Insert режим по-дълго от нужното. Vim е проектиран така, че много операции (навигация, търсене, копиране) са по-бързи в Normal режим. Обичайният работен поток е: влизаш в Insert, пишеш конкретна промяна, излизаш в Normal.

## Режим Visual

**Visual режимът** позволява да **избереш блок от текст** и да приложиш команди върху него. Работи подобно на маркирането с мишката, но изцяло с клавиатурата.

```
v             →   Visual — избира символ по символ
V             →   Visual Line — избира цели редове
Ctrl+v        →   Visual Block — избира правоъгълен блок (колони)
```

Докато си в Visual режим, движиш курсора за да разшириш или свиеш избора, а после прилагаш команда:

```
d             →   изтрива избраното
y             →   копира избраното
c             →   изтрива избраното и влиза в Insert
>             →   увеличава отстъпа (indent)
<             →   намалява отстъпа
=             →   автоматично форматира отстъпа
~             →   обръща главни/малки букви
:             →   влиза в Command режим само за избраните редове
```

### Visual Block — редактиране на колони

`Ctrl+v` е особено мощен — позволява да редактираш **вертикален блок** от текст едновременно:

```
Ctrl+v → избираш колона → I → пишеш текст → Esc
```

Така добавяш един и същ текст в началото на множество редове наведнъж.

```
Преди:                     След Ctrl+v, избор, I, "# ", Esc:
  line one                   # line one
  line two          →        # line two
  line three                 # line three
```

## Режим Command

**Command режимът** се активира с `:` от Normal режим. Той позволява да изпълняваш команди — запазване, затваряне, търсене и замяна, настройки и много повече. Командата се въвежда в долния ред на екрана и се изпълнява с `Enter`.

### Файлови операции

```
:w            →   запазва файла (write)
:w file.txt   →   запазва като file.txt
:q            →   затваря (quit) — само ако няма незапазени промени
:q!           →   затваря БЕЗ запазване (! = принудително)
:wq           →   запазва и затваря
:x            →   запазва и затваря (само ако има промени)
ZZ            →   запазва и затваря (в Normal режим, без :)
ZQ            →   затваря без запазване (в Normal режим)
```

### Намиране и заместване

Command режимът поддържа пълен `sed`-подобен синтаксис за търсене и замяна:

```
:s/foo/bar/          →   замества първото "foo" на текущия ред
:s/foo/bar/g         →   замества всички "foo" на текущия ред
:%s/foo/bar/g        →   замества всички "foo" в целия файл
:%s/foo/bar/gc       →   замества с потвърждение за всяко (c = confirm)
:5,10s/foo/bar/g     →   само на редове 5 до 10
:'<,'>s/foo/bar/g    →   само в Visual selection (попълва се автоматично)
```

### Навигация и преглед

```
:42           →   отива на ред 42
:set number   →   показва номерата на редовете
:set nonumber →   скрива номерата
:set list     →   показва скрити символи (tabs, trailing spaces)
:syntax on    →   включва синтактично оцветяване
```

### Работа с множество файлове

```
:e file.txt   →   отваря file.txt в текущия буфер
:split        →   разделя прозореца хоризонтално
:vsplit       →   разделя прозореца вертикално
:bn           →   следващ буфер (buffer next)
:bp           →   предишен буфер (buffer previous)
:ls           →   показва всички отворени буфери
```

## Какво е операционна система?

Компютърният хардуер сам по себе си не прави нищо. Процесорът, паметта и диска са просто физически компоненти — те не знаят какво означава "отвори файл" или "свържи се с интернет". Нужен е посредник, който да преведе човешките намерения в инструкции за хардуера. Това е **операционната система (ОС)**.

Операционната система е софтуер, който:
- управлява хардуерните ресурси (процесор, памет, дискове, мрежа)
- позволява на програмите да работят, без да се налага всяка от тях да "говори" директно с хардуера
- осигурява обща среда — файлова система, потребители, права за достъп
- служи като арбитър — решава кой процес получава процесорно време и колко памет

Примери за операционни системи: **Linux**, **macOS**, **Windows**, **Android**, **iOS**.

## Слоевете

Полезно е да мислиш за компютъра като за стек от слоеве, като всеки слой говори само с тези директно над и под него:

```
┌─────────────────────────────────┐
│         Потребителят            │  ← ти
├─────────────────────────────────┤
│    Приложения (браузър, IDE…)   │  ← програмите, които ползваш
├─────────────────────────────────┤
│      Shell / Терминал           │  ← интерфейсът за управление
├─────────────────────────────────┤
│    Операционна система (ядро)   │  ← управлява всичко
├─────────────────────────────────┤
│         Хардуер                 │  ← CPU, RAM, диск, мрежа
└─────────────────────────────────┘
```

**Ядрото (kernel)** е най-дълбоката и критична част на ОС. То работи директно с хардуера и никоя програма не може да го заобиколи. Когато напишеш `ls`, терминалът говори с shell-а, shell-ът говори с ядрото, ядрото чете от диска и резултатът се връща нагоре по стека.

## Какво е терминалът?

В миналото компютрите са били огромни машини, до които са се свързвали чрез **физически терминали** — устройства с клавиатура и монитор, без собствен процесор. Терминалът е бил просто прозорец към централната машина.

Днес физическите терминали са история, но концепцията е останала. Съвременният **терминален емулатор** (Terminal.app, iTerm2, GNOME Terminal, Windows Terminal) е програма, която симулира старото поведение — дава ти текстов интерфейс, чрез който да комуникираш с операционната система.

> **Терминалът сам по себе си не изпълнява команди.** Той е просто прозорец — показва текст и предава натиснатите клавиши. Истинската работа се върши от **shell-а**, който работи вътре в него.

### GUI vs CLI

Съществуват два основни начина да взаимодействаш с операционната система:

**GUI (Graphical User Interface)** — прозорци, икони, мишка. Интуитивен и визуален, но ограничен — можеш да правиш само това, което разработчикът е предвидил в интерфейса.

**CLI (Command Line Interface)** — текстови команди в терминала. Изглежда сложно отначало, но е изключително мощен — можеш да автоматизираш задачи, да комбинираш команди, да работиш с отдалечени сървъри и да правиш неща, за които просто няма бутон в GUI.

```
GUI: Отваряш File Explorer → навигираш → влачиш файл → потвърждаваш
CLI: mv documents/report.pdf ~/Desktop/
```

Нито едното не е "по-добро" — те са различни инструменти. Повечето опитни разработчици използват и двете.

## Shell

**Shell-ът** е програмата, която чете командите ти, интерпретира ги и ги изпълнява. Той е преводачът между теб и ядрото на операционната система.

Когато напишеш `ls -la` и натиснеш Enter, ето какво се случва:

```
1. Терминалът предава текста на shell-а
2. Shell-ът разпознава "ls" като команда и "-la" като флагове
3. Shell-ът казва на ядрото: "прочети тази директория"
4. Ядрото чете от диска и връща данните
5. Shell-ът форматира резултата
6. Терминалът го показва на екрана
```

### Видове shell

Съществуват различни shell-ове — те имат едни и същи базови функции, но се различават по синтаксис и възможности:

| Shell | Пълно име | Употреба |
|-------|-----------|----------|
| `sh` | Bourne Shell | Оригиналният Unix shell — базов стандарт |
| `bash` | Bourne Again Shell | Най-разпространен, стандартен в Linux |
| `zsh` | Z Shell | Стандартен в macOS, много разширения |
| `fish` | Friendly Interactive Shell | Много удобен за начинаещи, автодопълване |
| `dash` | Debian Almquist Shell | Много бърз, ползва се за скриптове в системата |

За да видиш кой shell използваш:
```bash
echo $SHELL        # пълният път — напр. /bin/bash или /bin/zsh
echo $0            # само името на shell-а
```

### Промпт (подканата)

Когато отвориш терминал, виждаш **промпта** — текстът преди курсора, който те кани да пишеш. Изглежда различно в различните системи, но типично съдържа:

```
ivan@laptop:~/projects$
│     │       │       └── $ = обикновен потребител  (# = root)
│     │       └────────── текуща директория
│     └────────────────── име на машината (hostname)
└──────────────────────── потребителско ime
```

## Файловата система

### Всичко е файл

В Linux/Unix има едно фундаментално правило: **всичко е файл**. Не само документи и снимки — дисковете, клавиатурата, мрежовите връзки, работещите процеси — всичко е представено като файл някъде в системата. Това прави ОС изключително последователна и предсказуема.

### Дървото на директориите

Файловата система в Linux е едно дърво с корен `/` (root). Няма `C:\` или `D:\` — всичко е под `/`, включително другите дискове (те се "монтират" в поддиректории).

```
/
├── bin/        → основни изпълними програми (ls, cp, mv…)
├── etc/        → конфигурационни файлове на системата
├── home/       → домашните директории на потребителите
│   └── ivan/   → твоята директория (~)
├── var/        → променливи данни (логове, кеш, бази данни)
│   └── log/    → системни логове
├── tmp/        → временни файлове (изтриват се при рестарт)
├── usr/        → програми и библиотеки инсталирани от потребителя
│   ├── bin/    → изпълними файлове на програмите
│   └── lib/    → библиотеки
├── dev/        → устройства (дискове, терминали…)
├── proc/       → информация за работещите процеси (виртуална ФС)
└── root/       → домашна директория на root потребителя
```

### Специални директории

| Символ | Означава |
|--------|----------|
| `/` | коренът на файловата система |
| `~` | домашната директория на текущия потребител (`/home/ivan`) |
| `.` | текущата директория |
| `..` | директорията едно ниво нагоре |
| `-` | предишната директория (само при `cd`) |

## Процеси

Всяка програма, която работи на компютъра, е **процес**. Когато стартираш браузъра, операционната система създава нов процес с уникален идентификатор — **PID (Process ID)**. Ядрото управлява всички процеси — разпределя им процесорно време, памет и достъп до ресурси.

```bash
ps aux              # показва всички работещи процеси
top                 # интерактивен преглед в реално време
htop                # по-удобна версия на top (може да се наложи инсталация)
kill 1234           # изпраща сигнал за спиране до процес с PID 1234
kill -9 1234        # принудително спиране (когато нормалното не работи)
```

Всеки процес има **родител** — процесът, който го е стартирал. Когато напишеш команда в терминала, shell-ът е родителят на новия процес. Ако затвориш терминала, процесите в него получават сигнал за спиране.

Процесите могат да работят:
- **Foreground** — в предния план, заемат терминала докато свършат
- **Background** — във фонов режим, освобождават терминала веднага

```bash
./script.sh           # foreground — чакаш да свърши
./script.sh &         # background — веднага получаваш промпта обратно
Ctrl+C                # прекъсва foreground процес
Ctrl+Z                # спира (suspend) foreground процес временно
bg                    # изпраща спрения процес в background
fg                    # връща background процес в foreground
```

## Потребители и права

Linux е многопотребителска система — проектирана да работи с много хора едновременно, всеки с различни права.

### Root потребителят

**root** е суперпотребителят — има неограничени права над цялата система. Може да чете, пише и изтрива всичко. Точно заради това **не работиш като root при нормална употреба** — грешка или злонамерен код могат да унищожат системата без никакво предупреждение.

Вместо това, когато имаш нужда от административни права за конкретна команда, използваш `sudo` (superuser do):

```bash
sudo apt install nginx      # инсталира програма с root права
sudo systemctl restart ssh  # рестартира системна услуга
sudo -i                     # отваря цял shell като root (внимавай)
```

`sudo` изисква паролата ти (не на root), проверява дали имаш право да го използваш, и логва всяка команда — това е важно от гледна точка на сигурността.

### Права за достъп

Всеки файл има права за три групи: **собственик (user)**, **група (group)** и **всички останали (other)**. За всяка група се дефинират права за четене (`r`), писане (`w`) и изпълнение (`x`).

```bash
ls -l file.txt
# -rw-r--r-- 1 ivan developers 1024 Mar 10 file.txt
#  │││││││││   │    └── група
#  │││││││││   └─────── собственик
#  ││├─┤├─┤├─┤
#  ││ u  g  o
#  │└──────────── тип: - файл, d директория, l symlink
```

Правата се задават с `chmod`, собственикът се сменя с `chown` — подробно разгледано в Седмица 1.

## Чести въпроси

**Защо терминалът изглежда толкова страшно?**
Защото е непознат, не защото е сложен. Графичният интерфейс е по-интуитивен, защото е проектиран да бъде — показва ти опциите визуално. Терминалът изисква да знаеш какво искаш, но след като се научиш, е многократно по-бърз и мощен.

**Мога ли да счупя системата с терминала?**
Да, по-лесно отколкото с GUI — затова се учи внимателно. Командите като `rm -rf` не питат за потвърждение и нямат кошче. Основното правило: **никога не копирай команда от интернет без да разбереш какво прави**, особено ако изисква `sudo`.

**Каква е разликата между Linux и Unix?**
Unix е оригиналната операционна система, създадена в Bell Labs през 70-те. Linux е отделно, свободно (open-source) ядро, вдъхновено от Unix но написано от нулата от Линус Торвалдс през 1991 г. macOS е базиран на BSD — друг наследник на Unix. Всички те споделят общата Unix философия и затова командите им са толкова сходни.

**Каква е разликата между Linux и Ubuntu/Debian/Fedora?**
**Linux** е само ядрото — управлява хардуера, но само по себе си не е пълна ОС. **Дистрибуциите** (Ubuntu, Debian, Fedora, Arch…) комбинират ядрото с графична среда, пакетен мениджър, системни инструменти и стандартни програми, за да получиш работещ компютър. Ядрото е едно и също — разликата е в "обвивката" около него.

**Защо командите в macOS и Linux са почти еднакви?**
Защото и двете системи следват стандарта **POSIX** — набор от правила, дефиниращи как трябва да изглеждат базовите инструменти в Unix-подобна ОС. `ls`, `cd`, `chmod`, `find` работят почти идентично на двете системи.

**Какво е PATH?**
Когато напишеш `ls`, shell-ът трябва да намери файла `/bin/ls`. `PATH` е списък от директории, в които shell-ът търси изпълними файлове. Ако инсталираш нова програма и тя "не се вижда" от терминала, причината почти винаги е, че директорията й не е в `PATH`.

```bash
echo $PATH
# /usr/local/bin:/usr/bin:/bin:/home/ivan/.local/bin
```

**Какво означава `$` пред думите като `$PATH` и `$SHELL`?**
`$` означава "стойността на тази променлива". `PATH`, `SHELL`, `HOME` са **environment variables** — системни променливи, достъпни за всички процеси. `$PATH` не е буквално "PATH", а стойността, която се пази под това име.

```bash
echo $HOME      # /home/ivan
echo $USER      # ivan
echo $PATH      # всички директории в пътя
```

**Защо понякога трябва да напиша `./script.sh` вместо просто `script.sh`?**
По сигурностни причини текущата директория (`.`) не е в `PATH` по подразбиране. Ако беше, злонамерен файл с имe `ls` в текущата директория щеше да се изпълни вместо истинския `ls`. `./` изрично казва "стартирай този файл от тук, не го търси в PATH".

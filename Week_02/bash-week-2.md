## `cat` — Concatenate

`cat` чете един или повече файла и отпечатва съдържанието им в терминала. Въпреки простото си предназначение, използва се изключително често — за бързо преглеждане на файлове, комбиниране на съдържание и пренасочване към други команди.

### Основна употреба

```bash
cat file.txt                        # показва съдържанието на файл
cat file1.txt file2.txt             # показва двата файла един след друг
cat file1.txt file2.txt > merged.txt  # обединява ги в нов файл
```

### Флагове

| Флаг | Значение |
|------|----------|
| `-n` | номерира всички редове |
| `-b` | номерира само непразните редове |
| `-s` | свива няколко последователни празни реда в един |
| `-A` | показва специални символи (`$` в края на ред, `^I` за tab) |
| `-E` | показва `$` в края на всеки ред |
| `-T` | показва табулациите като `^I` |
| `-v` | показва непечатаемите символи |

```bash
cat -n script.sh          # номерирани редове — полезно при дебъгване
cat -b notes.txt          # само непразните редове получават номер
cat -s log.txt            # премахва излишните празни редове
cat -A config.txt         # открива скрити символи (trailing spaces, tabs)
```

### Създаване на файл директно от терминала

`cat` може да се използва и за писане на съдържание директно в нов файл — въвеждаш текст ред по ред, и приключваш с `Ctrl+D`:

```bash
cat > newfile.txt
Това е първи ред.
Това е втори ред.
^D                        # Ctrl+D — край на входа
```

За да добавиш към съществуващ файл (без да го презаписваш), използвай `>>`:
```bash
cat >> existing.txt
Този ред се добавя накрая.
^D
```

### Пренасочване и pipe

`cat` е много удобен за подаване на съдържание към други команди чрез `|` (pipe):

```bash
cat file.txt | grep "error"        # търси думата "error" в файла
cat file.txt | wc -l               # брои редовете
cat file.txt | sort                # сортира редовете
cat access.log | grep "404" | wc -l  # колко 404 грешки има в лога
```

> **Съвет:** За много дълги файлове е по-удобно да използваш `less` или `more` вместо `cat`, за да можеш да скролваш.

## `man` — Manual

`man` показва вградената документация (наръчника) за всяка команда в системата. Това е първото място, където трябва да погледнеш, когато не знаеш как работи дадена команда или искаш да видиш всички й флагове.

```bash
man ls         # отваря наръчника за ls
man chmod      # отваря наръчника за chmod
man man        # наръчникът за самия man
```

### Как изглежда една man страница

Всяка man страница е разделена на стандартни секции:

| Секция | Съдържание |
|--------|------------|
| **NAME** | Кратко описание на командата |
| **SYNOPSIS** | Синтаксисът — как се извиква командата |
| **DESCRIPTION** | Подробно описание на поведението |
| **OPTIONS** | Всички флагове с обяснения |
| **EXAMPLES** | Примери за употреба (не винаги присъства) |
| **SEE ALSO** | Свързани команди |
| **BUGS** | Известни проблеми (рядко) |
| **AUTHOR** | Авторът на командата/документацията |

### Навигация в man страницата

`man` използва програмата `less` за показване на текста, затова навигацията е същата:

| Клавиш | Действие |
|--------|----------|
| `Space` / `f` | страница напред |
| `b` | страница назад |
| `↑` / `↓` | ред нагоре/надолу |
| `g` | към началото |
| `G` | към края |
| `/търсена_дума` | търсене напред |
| `?търсена_дума` | търсене назад |
| `n` | следващо съвпадение |
| `N` | предишно съвпадение |
| `q` | изход |

### Секции на наръчника

`man` е организиран в 9 секции — всяка покрива различен тип документация:

| Секция | Съдържание |
|--------|------------|
| 1 | Потребителски команди (ls, cat, chmod…) |
| 2 | Системни извиквания (open, read, write…) |
| 3 | C библиотечни функции (printf, malloc…) |
| 4 | Специални файлове (`/dev/null`, `/dev/sda`…) |
| 5 | Формати на конфигурационни файлове (fstab, passwd…) |
| 6 | Игри и занимателни програми |
| 7 | Различни концепции (regex, signal, ascii…) |
| 8 | Системни администраторски команди (mount, fsck…) |
| 9 | Документация на ядрото (рядко използвана) |

Когато едно и също име съществува в няколко секции, задаваш секцията изрично:

```bash
man 1 printf      # командата printf в bash
man 3 printf      # функцията printf в C
man 5 passwd      # форматът на файла /etc/passwd
man 8 mount       # командата mount за администратори
```

### Флагове

| Флаг | Значение |
|------|----------|
| `-k` | търси по ключова дума в имената и описанията (`apropos`) |
| `-f` | показва кратко описание на командата (`whatis`) |
| `-a` | показва всички man страници с това име (от всички секции) |
| `-w` | показва пътя до файла на man страницата |
| `--no-hyphenation` | изключва пренасянето на думи |

```bash
man -k "copy file"      # търси команди, свързани с копиране на файлове
man -f ls               # кратко: "ls (1) - list directory contents"
man -a printf           # показва всички секции за printf
man -w chmod            # /usr/share/man/man1/chmod.1.gz
```

### Полезни алтернативи

Ако man страницата е твърде дълга или тежка за четене, има удобни алтернативи:

```bash
tldr ls           # кратки практични примери (изисква инсталация: sudo apt install tldr)
ls --help         # вградена помощ директно от командата
info ls           # по-подробна документация в GNU формат
```

> **Съвет:** `man -k <ключова_дума>` е изключително полезен когато не помниш точното име на командата — търси из всички описания и ти показва кандидатите.

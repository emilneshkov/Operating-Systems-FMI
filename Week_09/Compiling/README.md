# Компилиране с Makefile

## Какво е Makefile

`make` е инструмент, който автоматизира компилирането. Вместо да пишеш дълга `gcc` команда всеки път, описваш правилата веднъж в `Makefile` и после просто пишеш `make`.

## Употреба

```
project/
├── Makefile
├── main.c
└── main         ← генерира се след make
```

```bash
make             # компилирай (генерира ./main)
make clean       # изтрий компилираните файлове
./main           # стартирай програмата
```

> **Забележка:** В рамките на този курс всички C програми се компилират по този начин — чрез предоставения `Makefile`. Не се очаква да пишеш `gcc` команди ръчно. Достатъчно е да сложиш `Makefile` и своя `.c` файл в една директория и да изпълниш `make`.

---

## Съдържание на Makefile-а

```makefile
#!/usr/bin/make -f
```

Shebang — казва на системата да използва `make` за изпълнение на файла.

---

### Компилатор

```makefile
CC= gcc
```

Дефинира компилатора. `CC` е стандартното име за "C Compiler". Ако искаш да смениш на `clang`, само тук го промяваш.

---

### CFLAGS — флагове за компилатора

```makefile
CFLAGS_OPT?=    -O2 -g -pipe
CFLAGS_STD?=    -std=c11
CFLAGS_WARN?=   -Wall -W -Wextra -Wpedantic
```

| Флаг         | Описание                                                            |
| ------------ | ------------------------------------------------------------------- |
| `-O2`        | Ниво на оптимизация — компилаторът прави кода по-бърз               |
| `-g`         | Включи debug информация (нужна за `gdb`)                            |
| `-pipe`      | Използвай pipe-ове вместо temp файлове между стъпките на компилация |
| `-std=c11`   | Използвай стандарта C11                                             |
| `-Wall`      | Включи **всички** стандартни предупреждения                         |
| `-W -Wextra` | Допълнителни предупреждения                                         |
| `-Wpedantic` | Строго спазване на стандарта                                        |

```makefile
CFLAGS?=    ${CFLAGS_OPT}
CFLAGS+=    ${CFLAGS_STD} ${CFLAGS_WARN}
```

`?=` — задай стойността **само ако променливата не е вече зададена** (може да се override-не от командния ред).  
`+=` — **добави** към съществуващата стойност.

---

### Допълнителни предупреждения

```makefile
#CFLAGS+=   -Werror
CFLAGS+=    -Wbad-function-cast -Wcast-align -Wcast-qual -Wchar-subscripts \
            -Winline -Wmissing-prototypes -Wnested-externs -Wpointer-arith \
            -Wredundant-decls -Wshadow -Wstrict-prototypes -Wwrite-strings
```

Разширен набор от предупреждения за хващане на типични C грешки:

| Флаг                   | Хваща                                             |
| ---------------------- | ------------------------------------------------- |
| `-Wbad-function-cast`  | Cast на функция към несъвместим тип               |
| `-Wcast-align`         | Cast, който нарушава alignment изисквания         |
| `-Wmissing-prototypes` | Функция без предварителна декларация              |
| `-Wshadow`             | Локална променлива скрива външна с същото ime     |
| `-Wwrite-strings`      | Запис в string литерал                            |
| `-Wstrict-prototypes`  | Функция без изрично деклариран тип на параметрите |

`-Werror` (закоментиран) би превърнал всяко предупреждение в **грешка** — компилацията спира. Добра идея за продукционен код.

`\` в края на ред означава продължение на следващия ред — чисто форматиране.

---

### CPPFLAGS — флагове за препроцесора

```makefile
CPPFLAGS_STD?=  -D_POSIX_C_SOURCE=200809L -D_XOPEN_SOURCE=700
CPPFLAGS+=      ${CPPFLAGS_STD}
```

`-D` дефинира макрос — еквивалентно на `#define` в кода. Тези две дефиниции отключват POSIX и X/Open функции в хедърите:

| Дефиниция                   | Отключва                                              |
| --------------------------- | ----------------------------------------------------- |
| `-D_POSIX_C_SOURCE=200809L` | POSIX.1-2008 функции: `pread`, `pwrite`, `getline`... |
| `-D_XOPEN_SOURCE=700`       | X/Open (XSI) разширения на POSIX                      |

Без тях функции като `getline()` или `strdup()` може да не са видими, дори да са в glibc.

---

### Файлове

```makefile
PROG=   main       # ime на изпълнимия файл
OBJS=   main.o     # object файлове (.o)
SRCS=   main.c     # изходен код (.c)
```

---

### Правила за компилация

```makefile
all:        ${PROG}
```

`all` е **default правилото** — изпълнява се при `make` без аргументи. Казва: "за да имаш `all`, трябва да имаш `${PROG}`".

```makefile
${PROG}:    ${OBJS}
            ${CC} ${LDFLAGS} -o ${PROG} ${OBJS}
```

За да се построи `main`, трябва `main.o`. Командата свързва (link-ва) object файловете в изпълним файл.

```makefile
${OBJS}:    ${SRCS}
            ${CC} ${CPPFLAGS} ${CFLAGS} -c -o ${OBJS} ${SRCS}
```

За да се построи `main.o`, трябва `main.c`. Командата компилира (без link) — флагът `-c` казва "само компилирай, не link-вай".

**Пълната команда, която реално се изпълнява:**

```bash
gcc -D_POSIX_C_SOURCE=200809L -D_XOPEN_SOURCE=700 \
    -O2 -g -pipe -std=c11 -Wall -W -Wextra -Wpedantic \
    -Wbad-function-cast ... -Wwrite-strings \
    -c -o main.o main.c

gcc -o main main.o
```

```makefile
clean:
        ${RM} -- ${OBJS} ${PROG}
```

`make clean` изтрива `main.o` и `main`. `--` означава "край на флагове" — предпазва от файлове с имена като `-f`.

---

### `.PHONY`

```makefile
.PHONY: all clean
```

Казва на `make` че `all` и `clean` са **имена на правила, не на файлове**. Без това, ако случайно имаш файл на ime `clean`, `make clean` нямаше да направи нищо (щеше да счита файла за актуален).

---

## Как make решава какво да компилира

`make` сравнява **времето на последна промяна** на файловете:

```
main.c  (променен в 10:00)
main.o  (създаден в 09:00)  ← по-стар → трябва прекомпилация
main    (създаден в 09:01)  ← по-стар → трябва re-link
```

Ако `main.c` не е променян след последното `make` — нищо не се прекомпилира:

```bash
make        # компилира всичко
make        # Nothing to be done for 'all'.
touch main.c
make        # компилира отново — main.c е по-нов от main.o
```

---

## Бърза справка

```bash
make              # компилирай (стартира правило 'all')
make clean        # изтрий компилираните файлове
make clean all    # изтрий и компилирай наново

# Override на променлива от командния ред
make CC=clang     # компилирай с clang вместо gcc
make CFLAGS_OPT="-O0 -g"  # без оптимизация, само debug
```

---

_За проект с един `.c` файл: сложи `Makefile` и `main.c` в една директория и пиши `make`._

## Пътища

**Абсолютният път** започва винаги от корена на файловата система — `/` — и не зависи от това къде се намираш в момента.

```bash
/home/ivan/documents/notes.txt
/etc/nginx/nginx.conf
```

**Релативният път** започва спрямо текущата директория и не започва с `/`.

```bash
documents/notes.txt       # директория documents вътре в текущата
./script.sh               # . означава "тук, в текущата директория"
../other-folder/file.txt  # .. означава "една директория нагоре"
```

> **Аналогия:** Абсолютният път е като пълен адрес (`гр. София, ул. Витоша 5`), а релативният е като „завий надясно след светофара".

## Основни команди

### `pwd` — Print Working Directory
Показва пълния (абсолютен) път до текущата директория.
```bash
pwd
# /home/ivan/projects
```

---

### `ls` — List
Показва съдържанието на директория. Може да приема конкретен път или да работи с текущата директория.
```bash
ls                  # списък на текущата директория
ls /home/ivan       # списък на конкретна директория
```

| Флаг | Значение |
|------|----------|
| `-l` | дълъг формат (права, собственик, размер, дата) |
| `-a` | показва скритите файлове (започващи с `.`) |
| `-h` | размерите в четим формат (KB, MB, GB) — използва се с `-l` |
| `-R` | рекурсивно — показва и поддиректориите |
| `-t` | сортира по дата на промяна |
| `-r` | обратен ред на сортиране |

```bash
ls -la      # скрити файлове + дълъг формат
ls -lh      # дълъг формат с четими размери
ls -ltr     # най-старите файлове най-отгоре
```

---

### `cd` — Change Directory
Смяна на текущата директория. Без аргумент — отива директно в home.
```bash
cd /home/ivan    # абсолютен път
cd documents     # релативен път
cd ..            # една директория нагоре
cd ../..         # две директории нагоре
cd ~             # към home директорията на потребителя
cd -             # обратно към предишната директория
```

---

### `mkdir` — Make Directory
Създава нова директория. С флага `-p` създава цялото дърво наведнъж, без грешка ако директорията вече съществува.
```bash
mkdir projects                  # създава директория
mkdir -p projects/web/css       # създава цялото дърво наведнъж
mkdir -m 755 secret             # създава с конкретни права
```

| Флаг | Значение |
|------|----------|
| `-p` | създава и липсващите родителски директории |
| `-m` | задава права при създаването |
| `-v` | verbose — отпечатва всяка създадена директория |

---

### `touch`
Създава нов празен файл, или обновява времето на последна промяна на вече съществуващ.
```bash
touch notes.txt                   # създава файл (ако не съществува)
touch file1.txt file2.txt         # създава няколко файла наведнъж
touch -t 202401011200 old.txt     # задава конкретна дата/час
```

---

### `rm` — Remove
Изтрива файлове и директории. **Внимание: няма кошче!**
```bash
rm file.txt                 # изтрива файл
rm file1.txt file2.txt      # изтрива няколко файла
```

| Флаг | Значение |
|------|----------|
| `-r` | рекурсивно — изтрива директория и съдържанието й |
| `-f` | force — без питане за потвърждение |
| `-i` | интерактивен режим — пита за всеки файл |
| `-v` | verbose — показва какво изтрива |

```bash
rm -r old_project/     # изтрива директория рекурсивно
rm -rf temp/           # ⚠️ изтрива без въпроси — внимавай!
rm -ri folder/         # пита за всеки файл
```

---

### `rmdir` — Remove Directory
Изтрива **само празни** директории. Ако директорията не е празна — използвай `rm -r`.
```bash
rmdir empty_folder     # работи само ако е празна
rmdir -p a/b/c         # изтрива цялото дърво (ако всичко е празно)
```

---

### `cp` — Copy
Копира файлове и директории.
```bash
cp file.txt backup.txt           # копира файл
cp file.txt /home/ivan/docs/     # копира в друга директория
```

| Флаг | Значение |
|------|----------|
| `-r` | рекурсивно — копира директория с цялото съдържание |
| `-i` | пита преди презаписване |
| `-u` | копира само ако изходният файл е по-нов |
| `-v` | verbose — показва какво копира |
| `-p` | запазва правата, собственика и датата |

```bash
cp -r projects/ backup/projects/    # копира цяла директория
cp -rp data/ data_backup/           # копира с оригинални метаданни
```

---

### `mv` — Move
Мести файлове/директории. Използва се и за **преименуване**.
```bash
mv old_name.txt new_name.txt      # преименуване
mv file.txt /home/ivan/docs/      # местене в друга директория
mv dir1/ /tmp/                    # местене на директория
```

| Флаг | Значение |
|------|----------|
| `-i` | пита преди презаписване |
| `-u` | мести само ако изходният е по-нов |
| `-v` | verbose — показва операцията |

---

### `tree`
Показва структурата на директориите визуално като дърво. Ако не е инсталиран: `sudo apt install tree`
```bash
tree                  # текущата директория
tree /home/ivan       # конкретна директория
```

| Флаг | Значение |
|------|----------|
| `-L 2` | ограничава дълбочината до 2 нива |
| `-a` | показва скрити файлове |
| `-d` | само директории (без файлове) |
| `-f` | показва пълния път |
| `--dirsfirst` | директориите първи |

```bash
tree -L 2 -d      # само директории, 2 нива надолу
tree -a -L 1      # скрити файлове, само 1 ниво
```

**Примерен изход:**
```
.
├── documents
│   ├── notes.txt
│   └── report.pdf
├── projects
│   └── web
│       └── index.html
└── README.md
```

## chmod

`chmod` (**ch**ange **mod**e) задава права за достъп до файлове и директории.

### Типове права

| Символ | Право | Значение |
|--------|-------|----------|
| `r` | read | четене |
| `w` | write | писане / промяна |
| `x` | execute | изпълнение (за файлове) / влизане (за директории) |
| `-` | — | няма право |

### Три групи потребители

Всеки файл има права за три групи, показани в следния формат:
```
d  rwx  r-x  r-x
│   │    │    └── other  — всички останали потребители
│   │    └─────── group  — членове на групата-собственик
│   └──────────── user   — собственикът на файла
└──────────────── тип: d=директория, -=файл, l=символна връзка
```

- **user (u)** — потребителят, притежаващ файла; **group (g)** — група от потребители; **other (o)** — всички останали; **all (a)** — всички заедно.

### Octal Notation (двоично представяне)

Всяко право се кодира с 1 бит, а правата за всяка група са сума от отделните стойности:

| Символи | Бинарен | Octal |
|---------|---------|-------|
| `rwx` | `111` | 7 |
| `rw-` | `110` | 6 |
| `r-x` | `101` | 5 |
| `r--` | `100` | 4 |
| `-wx` | `011` | 3 |
| `-w-` | `010` | 2 |
| `--x` | `001` | 1 |
| `---` | `000` | 0 |

**Пример:**
```bash
chmod 755 script.sh   # 7=rwx (собственик), 5=r-x (група), 5=r-x (останали)
chmod 644 notes.txt   # 6=rw- (собственик), 4=r-- (група), 4=r-- (останали)
```

### Символен синтаксис

```bash
chmod u+x script.sh           # добавя право за изпълнение на собственика
chmod g-w file.txt            # премахва право за писане от групата
chmod o=r file.txt            # задава само четене за others
chmod a+x script.sh           # добавя изпълнение за всички
chmod u=rwx,g=rx,o=r f.txt    # задава всичко наведнъж
```

| Оператор | Значение |
|----------|----------|
| `+` | добавя право |
| `-` | премахва право |
| `=` | задава точно тези права (останалите се нулират) |

### Чести примери

```bash
chmod 755 script.sh    # изпълним скрипт — стандарт
chmod 644 config.txt   # конфигурационен файл
chmod 600 secret.key   # само собственикът чете/пише (лични ключове)
chmod 777 shared/      # ⚠️ всички имат всички права (избягвай)
chmod -R 755 project/  # рекурсивно за цяла директория
chmod +x deploy.sh     # бързо добавяне на изпълнение
```

> **Флагът `-R`** прилага правата рекурсивно — към директорията и всичко вътре в нея.

### Как да видиш правата

```bash
ls -l file.txt
# -rw-r--r-- 1 ivan developers 1024 Mar 10 14:22 file.txt
#  ││ │  └── other: r--
#  ││ └───── group: r--
#  │└──────── user:  rw-
#  └───────── тип файл (- = обикновен файл)
```

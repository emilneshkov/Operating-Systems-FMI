# tmux — Terminal Multiplexer

*[Защо използваме tmux](#why-tmux)*

## Какво е tmux?

Когато отвориш терминал, получаваш един прозорец с един шел. Ако искаш да правиш две неща едновременно — трябва да отвориш втори прозорец. `tmux` решава това по елегантен начин: той позволява да имаш **множество терминали в един прозорец**, организирани в сесии, прозорци и панели.

Но истинската му суперсила е друга — **сесиите в tmux продължават да работят дори след като затвориш терминала**. Ако се свържеш по SSH към сървър, стартираш tmux и пуснеш дълъг процес — можеш да затвориш лаптопа, да се прибереш, и на следващия ден да се reconnectнеш и да намериш всичко точно там, където си го оставил.

---

## Концепции

tmux е организиран в три нива:

```
Session (сесия)
└── Window / Tab (прозорец)
    └── Pane (панел)
```

- **Session** — най-горното ниво. Една сесия може да съдържа много прозорци. Сесията продължава да живее дори без активна връзка.
- **Window** — като таб в браузър. Всеки прозорец заема целия екран и може да бъде разделен на панели.
- **Pane** — разделение на прозореца. Всеки панел е отделен терминал, работещ независимо.

## Prefix клавиш

Всички команди в tmux се предхождат от **prefix** — специална клавишна комбинация, която казва на tmux "следващият клавиш е за теб, не за терминала". По подразбиране prefix-ът е `Ctrl + B`.

```
Ctrl+B, след това D    ← правилно: prefix, после команда
Ctrl+B+D               ← грешно: не задържай всичко едновременно
```

> **Запомни:** Натискаш `Ctrl+B`, пускаш, и след това натискаш следващия клавиш.

## Сесии

Сесията е контейнерът на всичко. Стартираш tmux и автоматично влизаш в нова сесия.

```bash
tmux                            # стартира нова анонимна сесия
tmux new -s work                # нова сесия с име "work"
tmux ls                         # показва всички активни сесии
tmux attach -t work             # свързва се към сесия "work"
tmux attach                     # свързва се към последната сесия
tmux kill-session -t work       # затваря сесия "work"
```

| Клавиш | Действие |
|--------|----------|
| `Ctrl+B, D` | detach — излизаш от сесията (тя продължава да работи) |
| `Ctrl+B, $` | преименуваш текущата сесия |
| `Ctrl+B, S` | показва всички сесии (с preview) |

## Прозорци (Windows / Табове)

Прозорците са като табове — всеки заема целия екран. В долната лента на tmux се вижда списъкът с отворените прозорци.

| Клавиш | Действие |
|--------|----------|
| `Ctrl+B, C` | създава нов прозорец |
| `Ctrl+B, ,` | преименува текущия прозорец |
| `Ctrl+B, W` | показва списък с всички прозорци |
| `Ctrl+B, N` | следващ прозорец |
| `Ctrl+B, P` | предишен прозорец |
| `Ctrl+B, 0-9` | прескача директно към прозорец по номер |
| `Ctrl+B, &` | затваря текущия прозорец (пита за потвърждение) |

---

## Панели (Panes / Split Screen)

Панелите са разделения на текущия прозорец — всеки панел е самостоятелен терминал.

```
┌─────────────────┬─────────────────┐
│                 │                 │
│    Панел 1      │    Панел 2      │
│                 │                 │
│                 ├─────────────────┤
│                 │    Панел 3      │
└─────────────────┴─────────────────┘
```

| Клавиш | Действие |
|--------|----------|
| `Ctrl+B, %` | разделя вертикално (ляво/дясно) |
| `Ctrl+B, "` | разделя хоризонтално (горе/долу) |
| `Ctrl+B, стрелки` | преминава между панелите |
| `Ctrl+B, O` | превключва към следващия панел (по ред) |
| `Ctrl+B, ;` | превключва между последните два панела |
| `Ctrl+B, Z` | zoom — разгъва панела на цял екран (и обратно) |
| `Ctrl+B, X` | затваря текущия панел (пита за потвърждение) |
| `Ctrl+B, Q` | показва номерата на панелите за секунда |
| `Ctrl+B, {` | мести текущия панел наляво |
| `Ctrl+B, }` | мести текущия панел надясно |

За преоразмеряване на панел задържаш prefix и натискаш стрелка:
```
Ctrl+B, Ctrl+стрелка     # преоразмерява панела стъпка по стъпка
```

## Статус лентата

В долната част на екрана tmux показва статус лентата:

```
[work] 0:bash  1:server* 2:logs-         "hostname"  10:32 10-Mar-26
  │      │        │   │
  │      │        │   └── * = текущ прозорец, - = предишен
  │      └────────┴─────── прозорци с номера и имена
  └─────────────────────── име на сесията
```

### Защо използваме tmux

В курса използваме **tmux**, за да работим по-удобно в терминала и да виждаме едновременно задачите и тяхното решение.

Често ще разделяме терминала на **два прозореца (split screen)**. В единия прозорец ще отваряме директорията със задачите:
```
/srv/fmi-os/exercises
```
и ще преглеждаме условието на задачата, а в другия прозорец ще пишем и изпълняваме необходимите команди.

Това ни позволява да работим по-бързо, без постоянно да превключваме между различни терминали или файлове.

---

> **Съвет:** Ако работиш много с tmux, си струва да конфигурираш `~/.tmux.conf` — можеш да смениш prefix от `Ctrl+B` на `Ctrl+A` (по-удобно), да включиш mouse support и да подобриш визуалния стил. Но за начало стандартните настройки са напълно достатъчни.
